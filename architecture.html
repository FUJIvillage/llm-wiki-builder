LLM Wiki Builder Architecture

Karpathy's LLM Wiki Philosophy — Read → Ask → Write → Integrate

Frontend Layer

  • • React + TypeScript + Tailwind CSS
  • • Query Inbox: 優先度順に選択式回答
  • • Wiki Browser: Markdownビューア + Diffレビュー
  • • Tauri WebView でネイティブ実行

Backend Layer

  • • Tauri Commands (Rust) で FS / DB アクセス
  • • Query Engine: LLM prompting + ベクトル類似度
  • • Integration Engine: 回答ストック → Wiki差分生成
  • • LLM Client: OpenAI-compatible API 抽象化

Storage Layer

  • • SQLite: Query / Answer / WikiPage / IntegrationBatch
  • • File System: raw/ (監視対象) + wiki/ (Markdown出力)
  • • ローカルファースト: 全データはユーザーマシン内
  • • Git自動コミットで変更履歴を保全（任意）
